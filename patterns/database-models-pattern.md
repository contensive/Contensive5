
# Database Models Pattern

> All patterns and API reference: [Patterns Index](https://raw.githubusercontent.com/contensive/Contensive5/refs/heads/master/patterns/index.md)

## Overview

Contensive database models are C# classes that map to database tables and provide CRUD operations, caching, and type-safe access to record data. All models inherit from `DbBaseModel`, which provides a rich set of static and instance methods for creating, reading, updating, and deleting records.

Models live in `source/Models/Models/Db/` within the `Contensive.Models.Db` namespace.

## Architecture

### Inheritance Chain

```
DbBaseFieldsModel        (base fields present in every table: id, name, ccguid, active, etc.)
    └── DbBaseModel      (CRUD methods: create, save, delete, createList, etc.)
        └── YourModel    (custom properties + tableMetadata)
```

- **DbBaseFieldsModel** defines the fields that exist in every Contensive table (see Base Fields below)
- **DbBaseModel** provides all the generic static and instance methods for database operations
- **Your model class** inherits `DbBaseModel`, declares a `tableMetadata` static property, and adds properties for custom fields

### Base Fields (DbBaseFieldsModel)

Every Contensive table has these columns, automatically managed by the framework. You do not need to declare them in your model — they are inherited.

| Property | Type | Description |
|----------|------|-------------|
| `id` | `int` | Primary key (auto-increment) |
| `name` | `string` | Record name, used in lookups and dropdown lists |
| `ccguid` | `string` | Optional GUID, auto-generated by the model |
| `active` | `bool` | Can be used to soft-disable records |
| `contentControlId` | `int` | ID of the metadata record in ccContent |
| `createdBy` | `int?` | Foreign key to ccMembers (who created it) |
| `createKey` | `int` | Used when creating new records |
| `dateAdded` | `DateTime?` | When the record was created |
| `modifiedBy` | `int?` | Foreign key to ccMembers (last modifier) |
| `modifiedDate` | `DateTime?` | When last modified |
| `sortOrder` | `string` | Optional field for custom sorting |

## Creating a New Model

### Step 1: Create the Class File

Create a new `.cs` file in `source/Models/Models/Db/` named `{YourEntity}Model.cs`.

### Step 2: Declare the Class

Every model follows this pattern:

```csharp
namespace Contensive.Models.Db {
    //
    public class MyEntityModel : DbBaseModel {
        //
        //====================================================================================================
        /// <summary>
        /// table definition
        /// </summary>
        public static DbBaseTableMetadataModel tableMetadata { get; } = new DbBaseTableMetadataModel("content name", "tablename", "default", false);
        //
        //====================================================================================================
        // -- custom properties here
        public string title { get; set; }
        public int categoryId { get; set; }
        public bool isPublished { get; set; }
    }
}
```

### Step 3: Define Table Metadata

The `tableMetadata` static property is required. It tells the framework how to find the data.

```csharp
public static DbBaseTableMetadataModel tableMetadata { get; } = new DbBaseTableMetadataModel(
    contentName,        // Content definition name (matches CDef Name in collection XML)
    tableName,          // Database table name (lowercase)
    dataSourceName,     // Usually "default"
    nameFieldIsUnique   // true if the Name field has a unique constraint
);
```

**Constructor overloads:**

```csharp
// Full constructor
new DbBaseTableMetadataModel("my content", "mytable", "default", true)

// Without nameFieldIsUnique (defaults to false)
new DbBaseTableMetadataModel("my content", "mytable", "default")

// Content name + table name only (dataSource defaults to "default")
new DbBaseTableMetadataModel("my content", "mytable")

// Table name only (contentName = tableName, dataSource = "default")
new DbBaseTableMetadataModel("mytable")
```

**Key conventions:**
- `contentName` — matches the `Name` attribute on the corresponding `<CDef>` in the addon collection XML (e.g., `"groups"`, `"add-ons"`, `"people"`)
- `tableName` — the actual database table name, typically prefixed with `cc` for core tables (e.g., `"ccgroups"`, `"ccmembers"`)
- Set `nameFieldIsUnique = true` when the Name field should be unique (enables `createByUniqueName()`)

## Property Types

### Simple Types

Map directly to database column types:

| C# Type | Database Column | Example |
|---------|----------------|---------|
| `string` | nvarchar | `public string title { get; set; }` |
| `int` | int | `public int categoryId { get; set; }` |
| `int?` | int (nullable) | `public int? processInterval { get; set; }` |
| `bool` | bit | `public bool isPublished { get; set; }` |
| `DateTime?` | datetime (nullable) | `public DateTime? dateExpires { get; set; }` |
| `double` | float | `public double latitude { get; set; }` |

### File Field Types

For fields that store content in external files (the database column holds the filename):

| Type | Use Case | Example |
|------|----------|---------|
| `FieldTypeFile` | Binary files (images, downloads) | `public FieldTypeFile imageFile { get; set; }` |
| `FieldTypeTextFile` | Text content stored in a file | `public FieldTypeTextFile copyFilename { get; set; }` |
| `FieldTypeHTMLFile` | HTML content stored in a file | `public FieldTypeHTMLFile htmlContent { get; set; }` |
| `FieldTypeJavascriptFile` | JavaScript stored in a file | `public FieldTypeJavascriptFile jsFilename { get; set; }` |
| `FieldTypeCSSFile` | CSS stored in a file | `public FieldTypeCSSFile stylesFilename { get; set; }` |

File field types have these properties:
- `.filename` — the path to the file on CDN storage
- `.content` — the text content of the file (lazy-loaded on first read)
- `.contentUpdated` — set to true when content is modified; triggers file write on `save()`

### Foreign Key Fields

Foreign keys to other tables are simple `int` properties. The naming convention is the related table's singular name + `Id`:

```csharp
public int organizationId { get; set; }    // FK to organizations table
public int emailTemplateId { get; set; }   // FK to email templates table
public int collectionId { get; set; }      // FK to addon collections table
```

## Using DbBaseModel Methods

All methods are generic and use `<T>` where `T` is your model type.

### Creating / Loading Records

```csharp
// Load by ID
var record = DbBaseModel.create<MyEntityModel>(cp, recordId);

// Load by GUID
var record = DbBaseModel.create<MyEntityModel>(cp, recordGuid);

// Load by unique name (requires nameFieldIsUnique = true in tableMetadata)
var record = DbBaseModel.createByUniqueName<MyEntityModel>(cp, "Record Name");

// Create a new record with default values from content definition
var record = DbBaseModel.addDefault<MyEntityModel>(cp);
var record = DbBaseModel.addDefault<MyEntityModel>(cp, userId);
var record = DbBaseModel.addDefault<MyEntityModel>(cp, defaultValues, userId);

// Create a new empty record (no defaults applied)
var record = DbBaseModel.addEmpty<MyEntityModel>(cp);
var record = DbBaseModel.addEmpty<MyEntityModel>(cp, userId);
```

### Saving Records

```csharp
// Save (instance method, returns the record id)
record.name = "Updated Name";
record.title = "New Title";
int id = record.save(cp);

// Save with user tracking
record.save(cp, userId);

// Save with async option
record.save(cp, userId, asyncSave: true);
```

### Deleting Records

```csharp
// Delete by ID
DbBaseModel.delete<MyEntityModel>(cp, recordId);

// Delete by GUID
DbBaseModel.delete<MyEntityModel>(cp, recordGuid);

// Delete rows matching SQL criteria
DbBaseModel.deleteRows<MyEntityModel>(cp, "categoryId=5");
```

### Querying Lists

```csharp
// All records
var allRecords = DbBaseModel.createList<MyEntityModel>(cp);

// With SQL WHERE criteria
var filtered = DbBaseModel.createList<MyEntityModel>(cp, "isPublished=1");

// With criteria and sort order
var sorted = DbBaseModel.createList<MyEntityModel>(cp, "isPublished=1", "name asc");

// With paging
var paged = DbBaseModel.createList<MyEntityModel>(cp, "isPublished=1", "name asc", pageSize: 20, pageNumber: 1);

// Get first match
var first = DbBaseModel.createFirstOfList<MyEntityModel>(cp, "isPublished=1", "dateAdded desc");
```

```csharp

        /// </summary>

    }

```

### Simple Model with Properties

```csharp
// LayoutModel.cs
namespace Contensive.Models.Db {
    //
    public class LayoutModel : DbBaseModel {
        //
        //====================================================================================================
        /// <summary>
        /// table definition
        /// </summary>

        //====================================================================================================
```

### Rule Model (Many-to-Many Join Table)

Rule models represent join tables for many-to-many relationships. They hold foreign keys to both sides of the relationship.

```csharp
// MemberRuleModel.cs
using System;

namespace Contensive.Models.Db {
    //
    public class MemberRuleModel : DbBaseModel {
        //
        //====================================================================================================
        /// <summary>
        /// table definition
        /// </summary>
        public static DbBaseTableMetadataModel tableMetadata { get; } = new DbBaseTableMetadataModel("member rules", "ccmemberrules", "default", false);
        //
        //====================================================================================================
        public DateTime? dateExpires { get; set; }
        public int groupId { get; set; }
        public int memberId { get; set; }
        public int groupRoleId { get; set; }
    }
}
```

```csharp
// GroupRuleModel.cs
namespace Contensive.Models.Db {
    //
    public class GroupRuleModel : DbBaseModel {
        //
        //====================================================================================================
        /// <summary>
        /// table definition
        /// </summary>
        public static DbBaseTableMetadataModel tableMetadata { get; } = new DbBaseTableMetadataModel("group rules", "ccgrouprules", "default", false);
        //
        //====================================================================================================
        public bool allowAdd { get; set; }
        public bool allowDelete { get; set; }
        public int contentId { get; set; }
        public int groupId { get; set; }
    }
}
```

### Model with Custom Static Methods

Models can add custom query methods that build on the base `createList<T>()`:

```csharp
// GroupModel.cs
using Contensive.BaseClasses;
using System;
using System.Collections.Generic;

namespace Contensive.Models.Db {
    //
    public class GroupModel : DbBaseModel {
        //
        //====================================================================================================
        /// <summary>
        /// table definition
        /// </summary>
        public static DbBaseTableMetadataModel tableMetadata { get; } = new DbBaseTableMetadataModel("groups", "ccgroups", "default", true);
        //
        //====================================================================================================
        public bool allowBulkEmail { get; set; }
        public string caption { get; set; }
        public DbBaseModel.FieldTypeTextFile copyFilename { get; set; }
        public bool publicJoin { get; set; }
        //
        //====================================================================================================
        /// <summary>
        /// Verify a group exists by name. If so, verify the caption. If not, create the group.
        /// </summary>
        public static GroupModel verify(CPBaseClass cp, string groupName, string groupCaption, int userId, Dictionary<string, string> DefaultValues) {
            GroupModel result = createByUniqueName<GroupModel>(cp, groupName);
            if (result != null) {
                if (result.caption != groupCaption) {
                    result.caption = groupCaption;
                    result.save(cp, userId);
                }
            } else {
                result = addDefault<GroupModel>(cp, DefaultValues, userId);
                result.name = groupName;
                result.caption = groupCaption;
                result.save(cp, userId);
            }
            return result;
        }
    }
}
```

### Complex Model (AddonModel)

Large models with many properties, enums, and multiple custom static methods. See `source/Models/Models/Db/AddonModel.cs` for the full example. Key patterns demonstrated:

- Many property types (`string`, `bool`, `int`, `int?`, `DateTime?`, `FieldTypeJavascriptFile`, `FieldTypeCSSFile`)
- `[Obsolete]` attribute on deprecated properties
- Custom static methods using `createList<T>()` with SQL subqueries
- Associated enums defined alongside the model

## Corresponding Collection XML CDef

Every model needs a matching `<CDef>` in the addon collection XML to configure admin editing and create the database table on install. The CDef `Name` must match the model's `tableMetadata.contentName`, and the CDef `ContentTableName` must match `tableMetadata.tableNameLower`.

For example, for `GroupModel`:

```csharp
// Model
public static DbBaseTableMetadataModel tableMetadata { get; } = new DbBaseTableMetadataModel("groups", "ccgroups", "default", true);
```

```xml
<!-- Collection XML CDef -->
<CDef Name="groups" ContentTableName="ccgroups" ... >
    <Field Name="allowBulkEmail" FieldType="Boolean" ... />
    <Field Name="caption" FieldType="Text" ... />
    <Field Name="copyFilename" FieldType="FileText" ... />
    <Field Name="publicJoin" FieldType="Boolean" ... />
</CDef>
```

See [addon-collection-pattern.md](addon-collection-pattern.md) for full CDef documentation.

## Property-to-Field Type Mapping

| C# Property Type | CDef FieldType |
|-------------------|---------------|
| `string` | `Text` or `LongText` |
| `int` | `Integer` |
| `int` (foreign key) | `Lookup` (with `LookupContent` set) |
| `bool` | `Boolean` |
| `DateTime?` | `Date` |
| `double` | `Float` |
| `FieldTypeTextFile` | `FileText` |
| `FieldTypeHTMLFile` | `FileHTML` |
| `FieldTypeJavascriptFile` | `FileJavascript` |
| `FieldTypeCSSFile` | `FileCSS` |
| `FieldTypeFile` | `File` or `FileImage` |

## Naming Conventions

- **Class name**: `{Entity}Model` in PascalCase (e.g., `GroupModel`, `PersonModel`, `AddonCategoryModel`)
- **File name**: matches class name, `{Entity}Model.cs`
- **Namespace**: always `Contensive.Models.Db`
- **Table name**: lowercase, typically `cc` prefix for core tables (e.g., `ccgroups`, `ccmembers`)
- **Content name**: lowercase, human-readable (e.g., `"groups"`, `"people"`, `"add-on categories"`)
- **Properties**: camelCase (e.g., `allowBulkEmail`, `dateExpires`, `organizationId`)
- **Foreign key properties**: `{relatedEntity}Id` in camelCase (e.g., `groupId`, `emailTemplateId`)
- **Rule models** (join tables): `{Entity}RuleModel` with table name `cc{entity}rules`
- **tableMetadata**: always declared as `public static DbBaseTableMetadataModel tableMetadata { get; } = new DbBaseTableMetadataModel(...);`

## Steps to Add a New Model

1. **Create the model class** in `source/Models/Models/Db/{Entity}Model.cs`
2. **Inherit from `DbBaseModel`**
3. **Declare `tableMetadata`** with content name, table name, data source, and name uniqueness
4. **Add properties** for each custom database field using the appropriate C# types
5. **Create the corresponding CDef** in the addon collection XML (see [addon-collection-pattern.md](addon-collection-pattern.md))
6. **Add custom static methods** if the model needs specialized query patterns (optional)
7. **Build the Models project**: `dotnet build source/Models/Models.csproj`

## Caching Notes

- Model caching applies only to classes directly in `Contensive.Models.Db`, not derived classes
- Individual records are cached by ID (`cachePrefix.id.{id}`) and optionally by GUID or name
- Lists of models are not cached
- Calling `save()` automatically invalidates the cache for that record
- Cache names replace spaces with underscores (e.g., `"addon collections"` becomes `addon_collections`)
